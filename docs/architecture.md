# ClawManager 架构说明

## 概述

ClawManager 是一款运行在 Docker 环境中的虚拟桌面管理平台，采用微服务架构设计，通过 Docker Compose 进行编排部署。

## 组件职责

### MySQL 8.4.8

**角色**: 关系型数据库，存储所有业务数据

**数据**:
- 用户账户与认证信息
- 虚拟桌面实例元数据
- 权限与配额配置
- AI Gateway 审计日志

**持久化路径**: `${MYSQL_DATA_PATH}`（默认为 `/mnt/Storage1/clawmanager/mysql`）

**健康检查**: `mysqladmin ping`，间隔 10s

---

### MinIO (S3 Compatible)

**角色**: 对象存储服务，兼容 Amazon S3 API

**用途**:
- 虚拟桌面镜像存储
- 用户上传文件
- 备份与快照存储

**持久化路径**: `${MINIO_DATA_PATH}`（默认为 `/mnt/Storage1/clawmanager/minio`）

**端口**:
- API: `:9000`
- Console: `:9001`（浏览器管理界面）

---

### Skill Scanner

**角色**: 技能目录扫描服务，索引可用的 AI 技能

**功能**:
- 扫描配置的技能目录
- 提供技能注册与发现 API
- 返回技能元数据（名称、描述、API 端点）

**特点**: 无状态，无需持久化数据

---

### ClawManager 主应用

**角色**: 核心业务服务，提供 Web UI 和 REST API

**技术栈**:
- 后端: Go (Gin 框架)
- 前端: React 19
- 认证: JWT

**功能**:
- 用户管理（注册、登录、角色）
- 虚拟桌面生命周期管理（创建/启动/停止/删除）
- 运行时镜像管理
- 资源配额控制
- AI Gateway 集成（模型治理、审计追踪）

**健康检查**: HTTP `GET /`，间隔 30s

---

## 数据流向

```
用户浏览器 (HTTPS)
      │
      ▼
ClawManager :30443 (反向代理/负载均衡)
      │
      ├──► MySQL :3306       (元数据读写)
      ├──► MinIO :9000        (文件读写)
      └──► Skill Scanner :8000 (技能查询)
```

## 网络设计

容器间通信通过 `clawmanager-net` 桥接网络实现，隔离于宿主机网络。

对外暴露端口（宿主机监听）：
| 端口 | 服务 | 说明 |
|------|------|------|
| `30443` | ClawManager | 主应用 HTTPS 访问 |
| `9000` | MinIO API | S3 兼容存储 API |
| `9001` | MinIO Console | 存储管理界面 |
| `8000` | Skill Scanner | 技能扫描 API |

## 环境变量驱动设计

所有路径、端口、凭证均通过 `.env` 文件注入，容器本身无硬编码配置。

详见 [`.env.example`](../.env.example)
